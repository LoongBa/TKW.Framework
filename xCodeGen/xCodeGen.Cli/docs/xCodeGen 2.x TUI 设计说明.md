# xCodeGen 2.x TUI 设计说明 (v1.0)

## 1. 项目背景

 `xCodeGen` 是 `TKWF.Domain V4` 领域自治框架的核心工具。2.x 版本旨在通过 TUI（终端用户界面）将“元数据驱动”的开发过程可视化，解决从手动配置到“领域自治”过程中环境稳定性、连接回收及 AOP 契约同步的复杂性。

## 2. TUI 核心价值

- **可视化领域视图**：直接通过 `IProjectMetaContext` 解析并渲染领域拓扑，无需启动 Web 环境即可掌握业务概况。

- **AOP 配置直观化**：通过勾选特性（Attributes）自动同步 `Controller` 与其 `IAopContract` 契约，消除手动同步代码的低级错误。

- **Agent 协作桥梁**：作为生成 Agency MD 指南的交互入口，为 AI Agent 提供精准的“技能地图”预览。

## 3. Cli 交互

xCodeGen 是一款基于元数据驱动的代码生成工具。

### 使用格式

执行命令格式：`xCodeGen [command] [options]`

### 可用命令

1. 无输入命令：直接进入 TUI 交互操作界面，命令参数值作为默认值；
2. gen：执行代码生成操作，可生成服务、控制器等业务代码；
3. init：快速初始化项目基础脚手架；
4. agent：自动生成 AI Agent 工作所需的领域规范指南文档。

### 附加参数选项

1. `-j / -json [path]`：手动指定配置文件的存放路径；
2. `-v / --verbose`：开启详细日志模式，展示完整执行过程明细。

## 4. 界面布局规划 (Layout)

采用经典的 **左右双栏 + 底部状态栏** 结构：

- **左侧栏 (宽度 30%)**：
  
  - **历史项目配置**：最近打开的配置文件/项目路径，默认选中 -j 参数传入的配置文件。
  
  - **预览/编辑配置**：实时修改 `xCodeGen.json` 参数（如 `EnableInterfaceExtraction` 开关）。
  
  - **路径探索器**：轻量级树形结构，用于进入文件浏览器模式选择目标项目。

- **右侧主区域 (宽度 70%)**：
  
  - **仪表盘 (Dashboard)**：显示统计信息（实体数、服务数、控制器数）及编译健康度。
  
  - **元数据详情**：分类展示 `Entities`、`DataServices`、`Controllers`。
  
  - **文档预览**：内置 Markdown 渲染引擎，用于预览即将提供给 Agent 的 `Agency.md`。

## 5. 交互设计 (Interaction)

当前 V2 版本功能说明：

- 项目配置文件列表：
  
  - 默认将`-j`参数选择的配置文件列入，置顶并选中；
  
  - 上下选择历史项目配置，回车键确认选中；
  
  - 选中的项目配置，在左下方窗口显示配置的各项参数，并可进入编辑模式；
  
  - 对于选中的项目配置，仪表盘显示信息、功能按钮可用（否则不可用）；

- 选择配置文件：可通过浏览按钮，在右侧展示文件浏览器，切换位置选中配置文件。选中的配置文件进入“项目配置文件列表”，置顶并选中。

- 功能按钮和快捷键：

- - **生成模板**：对应`gen 命令`，全局快捷键`F5`，实时在右侧`Log`窗口输出生成进度。注意可切换参数，对应`-v`
  
  - **初始化脚手架**：对应`init命令`，全局快捷键`F6`，先弹出窗口确认是否继续（如果已经存在脚手架，提示是否覆盖模板）
    ——搭建 TUI 框架，调用空函数，后续填充逻辑
  
  - **生成`AgencyMD`**：对应`agent命令`——搭建 TUI 框架，调用空函数，后续填充逻辑。
  
  - **预览`AgencyMD`**：预览选中项目的文档——搭建 TUI 框架，调用空函数，后续填充逻辑。

未来 V3 版本功能展望：

- **穿透式编辑**：选中`Controller`弹出`Modal`窗口，支持快速增删`[AuthorityFilter]`或 `[Transactional]`。

- **AI 联动**：支持在 TUI 中输入自然语言描述，由 Agent 自动修改领域实体并触发代码生成。

- **工具联动**：调用`OpenCode`等第三方工具生成代码。
